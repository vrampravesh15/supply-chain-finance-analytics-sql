# Supply Chain Finance Analytics SQL

## Overview
This project uses SQL to build and analyze a supply chain and finance analytics environment for a hardware business. It combines schema design, business query development, and analytical insights across sales, forecasting, profitability, freight costs, and customer behavior.

## Problem Statement
Supply chain and finance data is often spread across multiple tables and business processes, making it difficult to analyze profitability, demand planning, customer value, and operational efficiency in one place.

## Objectives
- Design a structured SQL-based analytics environment
- Analyze sales, forecast accuracy, discounts, and profitability
- Identify cost drivers and market opportunities
- Support data-driven decision-making with advanced SQL queries

## Tools & Technologies
- MS SQL Server
- SQL Queries
- Joins and Aggregations
- Window Functions
- Stored Procedures
- Triggers
- User Defined Functions
- Star Schema Modeling

## Methodology
1. Designed fact and dimension tables
2. Loaded customer, product, sales, pricing, and forecast data
3. Wrote analytical SQL queries for multiple business scenarios
4. Applied ranking, lag/lead, and profitability logic
5. Interpreted outputs into business insights and recommendations

## Key Insights
- High-selling products are not always the most profitable
- Forecast accuracy varies by product and market maturity
- Freight and discount levels influence overall margin performance
- Seasonal spikes require better inventory alignment
- Direct channels can outperform traditional channels on margin

## Recommendations
- Improve forecast models using historical error patterns
- Revisit discount strategy for margin-sensitive products
- Optimize logistics in high-cost markets
- Align inventory planning with seasonal sales behavior
